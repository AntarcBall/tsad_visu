Section IV · Challenges of Classical Approaches

왜 classical approach는
현대 TSAD에서 자주 무너지는가

논문은 classical method를 버려야 할 유물로 다루지 않습니다. 오히려 어디까지는 유효하고, 어디서부터 급격히 깨지는지를 분명히 보여줍니다. 핵심은 modern multivariate time-series anomaly detection가 요구하는 것이 단순한 thresholding이나 handcrafted feature를 넘어, temporal dependency, inter-variable relation, label scarcity, nonstationarity를 동시에 다루는 표현 학습이라는 점입니다.

Classical taxonomy

논문이 짚는 다섯 가지 classical family

중요한 건 이름보다 각 family가 어떤 가정을 전제로 하는지입니다.

Time / Frequency domain analysis

signal processing 기반 특징량과 스펙트럼 변화를 사용합니다. 물리적 직관이 좋고 설명성이 높지만, feature engineering 의존이 큽니다.

  • 장점: domain rule과 결합하기 쉬움
  • 약점: 복잡한 multivariate dependency를 직접 표현하기 어려움
  • 언제 무너지는가: 정상/비정상 경계가 국소 특징만으로 안 보일 때

Statistical model

분포, 평균/분산, control limit, Gaussian 가정 등을 사용해 이상을 포착합니다. threshold가 명확할수록 강하지만, 분포가 흔들리면 빠르게 취약해집니다.

  • 장점: 구현이 단순하고 운영 설명이 쉬움
  • 약점: nonstationarity와 heavy-tail/noisy regime에 약함
  • 언제 무너지는가: 시간에 따라 정상 분포 자체가 바뀔 때

Distance-based model

유사도, 최근접 거리, DTW류 거리 개념으로 비정상성을 판정합니다. local shape 비교에는 강하지만, scale이 커질수록 계산량과 표현 한계가 동시에 커집니다.

  • 장점: 직관적인 anomaly score
  • 약점: high-dimensional distance concentration 문제
  • 언제 무너지는가: 변수 수가 많아지며 거리 개념이 흐려질 때

Predictive model

다음 값을 예측하고 residual을 anomaly signal로 씁니다. 직관은 좋지만, long-range temporal context와 multivariate relation을 단순 예측기로 모두 담기 어렵습니다.

  • 장점: early warning 해석과 잘 맞음
  • 약점: 잔차가 항상 anomaly를 뜻하지는 않음
  • 언제 무너지는가: 정상 패턴 자체가 다양한 복합 regime일 때

Clustering model

정상 cluster에서 벗어난 샘플을 이상으로 봅니다. 비교적 범용적이지만, 시계열 문맥을 명시적으로 갖지 않는 순간 collective anomaly를 놓치기 쉽습니다.

  • 장점: label scarcity 환경에서 baseline으로 쓸 수 있음
  • 약점: 순서 정보와 causal relation을 잃기 쉬움
  • 언제 무너지는가: point는 정상인데 segment 전체가 비정상일 때

MapReduce-style scaling

논문은 Fig. 4를 통해 classical pipeline이 scale-out을 시도하는 방식을 짚습니다. 하지만 계산을 병렬화해도 feature representation이 약하면 본질적 한계는 남습니다.

  • 장점: 대용량 처리량 확보
  • 약점: representation bottleneck은 그대로
  • 핵심 교훈: scale 문제와 modeling 문제를 분리해서 봐야 함

Why they break

현대 multivariate TSAD에서 classical 가정이 깨지는 네 가지 이유

1. 순서 정보가 feature보다 더 중요해진다

static outlier detection은 값의 크기나 위치 차이를 중심으로 보지만, time-series anomaly는 언제 나타났는지와 앞뒤 패턴과 어떤 관계인지가 훨씬 중요합니다. 그래서 point anomaly에는 먹히던 룰이 contextual / collective anomaly에서는 무너집니다.

2. 변수는 독립이 아니라 얽혀 있다

센서, actuator, network traffic, control process는 물리적·논리적으로 연결되어 있습니다. classical feature engineering은 이 연결을 사후적으로 요약하려 하지만, relation 자체를 학습하는 모델에 비해 한계가 큽니다.

3. 정상은 하나의 점이 아니라 moving target이다

nonstationarity와 concept drift 때문에 “정상 평균”이나 “정상 cluster”는 고정점이 아닙니다. 따라서 statistical bound나 distance threshold를 수동으로 유지하는 비용이 급격히 증가합니다.

4. 표현력 부족이 계산력 부족보다 먼저 온다

병렬 처리나 분산 처리로 속도를 보완해도, 중요한 정보를 표현하지 못하면 anomaly score 자체가 흔들립니다. 논문이 deep learning으로 넘어가는 이유가 여기 있습니다.

Bridge to deep learning

논문이 deep learning taxonomy로 넘어가는 논리적 다리

Sec. IV는 “classical methods are bad”가 아니라 “modern TSAD가 요구하는 표현이 달라졌다”는 선언입니다.

표현 학습으로 이동

handcrafted feature 대신 representation learning이 필요합니다. 그래야 복잡한 sequence pattern과 latent relation을 같이 담을 수 있습니다.

Unsupervised / semi-supervised 전제

label scarcity 때문에 대부분의 practical TSAD는 정상 패턴 학습에서 시작해야 합니다. 이것이 reconstruction / prediction / dissimilarity 기반 deep model의 출발점입니다.

세 축으로 재정렬

이후 논문은 모델을 architecture 이름보다 inter-correlation, temporal context, anomaly criteria라는 세 축으로 읽게 만듭니다. 이것이 Sec. V의 taxonomy로 이어집니다.

요약하면, classical approach의 한계는 곧 deep learning의 필요조건입니다. 문제는 “더 복잡한 모델이 좋은가?”가 아니라 “지금 이 데이터에서 어떤 관계와 문맥을 표현해야 하는가?”입니다.

이 페이지를 보고 나면 답할 수 있어야 하는 질문

  • 왜 point anomaly용 threshold가 collective anomaly에서 무력해지는가?
  • 왜 high-dimensional multivariate setting에서 distance와 clustering이 급격히 어려워지는가?
  • 왜 scale-out만으로 TSAD가 해결되지 않는가?
  • 왜 논문이 deep learning taxonomy를 세 축으로 다시 그리는가?
