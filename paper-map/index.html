I

Introduction

이상 탐지가 산업 자동화, IoT, fault diagnosis에서 왜 중요한지 문제의식을 세웁니다. 기존 리뷰가 model list에 머무른 반면, 이 논문은 analysis + guidelines까지 제공한다는 차별점을 먼저 선언합니다.

이 섹션을 읽어야 하는 사람: “왜 이 논문이 아직도 reference value가 있는가?”를 빠르게 설명해야 하는 발표자.

II

Background

point / contextual / collective anomaly와 temporality, dimensionality, nonstationarity, noise를 정의합니다. 여기서 이미 “값 하나가 아니라 문맥이 중요하다”는 TSAD의 핵심 난제가 드러납니다.

연결 페이지: Background · 핵심 asset: Fig. 1, Table 1

III

Industrial Applications

smart manufacturing, energy, cloud, structural health monitoring을 훑으며 anomaly detection의 목적이 얼마나 다양한지 보여줍니다. 여기서부터 이미 “one best model”이 성립하기 어렵다는 실무적 이유가 보입니다.

연결 페이지: Applications · 핵심 asset: Fig. 2, Fig. 3

IV

Challenges of Classical Approaches

label scarcity, multivariate complexity, variable relation modeling 한계를 짚으며 왜 deep learning이 들어오는지 연결합니다. 논문 흐름상 taxonomy로 가기 전 필수 다리 역할을 합니다.

연결 페이지: Classical Limitations · 읽기 포인트: classical baseline의 약점을 “완전히 무효”가 아니라 “어디서 깨지는가”로 이해할 것.

V

Deep Learning Taxonomy

논문의 핵심 contribution입니다. inter-correlation, temporal context, anomaly criteria의 세 축으로 모델을 재배열해, architecture 나열을 넘어 문제 구조로 읽게 만듭니다.

연결 페이지: Methods · 핵심 asset: Fig. 5–8, Table 2–3

VI

Comparative Reviews

SWaT, WADI, MSL 위에서 model family의 강약을 읽습니다. 중요한 건 순위표가 아니라 dataset dimension, relation strength, temporal dependency가 결과를 어떻게 바꾸는지 이해하는 것입니다.

연결 페이지: Benchmarks · 핵심 asset: Fig. 9–10, Table 4–6

VII

Guidelines for Practitioners

논문의 실무적 클라이맥스입니다. real-time vs early warning, sliding window vs incremental update, denoising, online update를 통해 “좋은 모델”이 아니라 “좋은 운영 선택”을 묻습니다.

연결 페이지: Guidelines · 핵심 asset: Fig. 11

VIII

Conclusion

결론은 단순합니다. deep learning은 강력하지만 만능이 아니며, use case와 dataset property를 읽는 능력이 model choice만큼 중요합니다. 이 메시지가 사이트 전체 UX의 anchor입니다.

발표 마무리 문장으로 그대로 쓰기 좋은 섹션.
