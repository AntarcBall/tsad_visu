Core concepts

문제 정의를 이루는 기본 어휘

Background / Paper Map / Benchmarks를 읽을 때 가장 자주 다시 찾게 되는 용어들입니다.

Anomaly detection

정상 패턴에서 벗어나는 observation이나 segment를 식별하는 작업입니다. time-series에서는 값 하나보다 맥락이 더 중요할 수 있습니다.

Time-series data

시간 순서가 핵심 의미를 가지는 데이터입니다. 과거-현재-미래의 관계가 그대로 모델 설계에 들어갑니다.

Multivariate time series

여러 센서·채널·변수가 동시에 기록되는 시계열입니다. 각 변수의 개별 변화뿐 아니라 변수 간 관계도 중요합니다.

Point anomaly

짧은 순간에 급격히 튀는 이상입니다. threshold 기반 탐지로 접근하기 쉬우나 context를 놓치면 오탐이 많아질 수 있습니다.

Contextual anomaly

수치 자체는 평범해도 현재 계절성·phase·운영 상태에서는 비정상인 경우입니다. temporal context modeling이 특히 중요합니다.

Collective anomaly

개별 point는 정상처럼 보여도 연속된 구간 전체가 비정상인 패턴입니다. segment-level reasoning과 window design이 필요합니다.

Temporality

시간 순서가 의미를 가진다는 성질입니다. 이전 상태를 얼마나 길게 기억하느냐가 모델 성능에 직접 연결됩니다.

Dimensionality

변수 수가 많아질수록 관계 표현과 계산량이 급증하는 문제입니다. WADI 같은 고차원 dataset에서 특히 중요한 축입니다.

Nonstationarity

시간이 지나면서 데이터 분포나 정상 패턴 자체가 바뀌는 성질입니다. drift 대응 전략이 없으면 false alarm이 늘어납니다.

Modeling terms

모델 구조를 읽을 때 필요한 용어

Inter-correlation

여러 변수 사이의 상호관계입니다. sensor/actuator가 물리적으로 연결된 시스템에서는 이 관계를 무시하기 어렵습니다.

Temporal dependency

현재 상태가 과거 시점의 패턴에 의존하는 정도입니다. long-range dependency를 어떻게 잡는지가 RNN/Transformer의 핵심 차이 중 하나입니다.

Dimensional reduction

많은 변수를 더 적은 latent factor로 압축하는 전략입니다. 계산량은 줄지만 root cause 해석력은 떨어질 수 있습니다.

2D matrix representation

covariance, similarity matrix처럼 변수 관계를 2차원 구조로 바꾸는 방식입니다. multivariate relation을 joint하게 반영할 때 유용합니다.

Graph neural network (GNN)

변수들을 node와 edge로 보고 관계 구조를 학습하는 모델입니다. relation-heavy industrial setting에서 특히 강점이 있습니다.

Graph attention

각 node가 어떤 이웃 node를 더 중요하게 봐야 하는지 attention score로 학습하는 방식입니다. root-cause 친화적인 해석을 돕습니다.

RNN / LSTM / GRU

시퀀스를 순차적으로 읽으며 hidden state에 기억을 축적하는 계열입니다. temporal dependency modeling에 강하지만 병렬성은 약합니다.

Dilated RNN

skip connection과 dilation으로 더 긴 time horizon을 적은 단계로 보게 만드는 RNN 변형입니다.

Temporal convolutional network (TCN)

causal convolution과 dilation을 활용해 긴 receptive field를 확보하는 CNN 계열입니다. 병렬 처리 이점이 있습니다.

ConvLSTM

LSTM 내부 연산을 convolution으로 바꿔 spatial + temporal 정보를 함께 다루는 hybrid 구조입니다.

Transformer / self-attention

sequence 전체 시점을 한 번에 참조하며 long-range dependency를 병렬로 학습하는 구조입니다.

Latent space

원본 입력을 압축해 표현한 내부 공간입니다. anomaly는 이 공간에서 normal cluster와 멀어지는 방식으로도 포착됩니다.

Scoring & metrics

이상 여부를 숫자로 만드는 용어

  • Anomaly score: 모델이 현재 상태가 얼마나 비정상인지 수치로 표현한 값입니다.
  • Threshold: anomaly score가 이 값을 넘으면 이상으로 판정합니다.
  • Adaptive threshold: 데이터 변화에 따라 threshold를 동적으로 조정하는 방식입니다.
  • Reconstruction error: 입력과 복원 결과의 차이입니다. AE/VAE/GAN 계열에서 자주 쓰입니다.
  • Prediction error: 예측한 다음 시점과 실제 관측값의 차이입니다. early warning과 잘 연결됩니다.
  • Dissimilarity: cluster나 distribution 중심에서 얼마나 멀리 떨어졌는지를 뜻합니다.
Evaluation metrics

논문 결과표를 읽는 지표

  • Precision: 탐지한 것 중 실제 anomaly의 비율입니다.
  • Recall: 실제 anomaly 중 모델이 잡아낸 비율입니다.
  • F1-score: precision과 recall의 균형 지표입니다.
  • False positive: 정상인데 anomaly로 잘못 판단한 경우입니다.
  • False negative: anomaly인데 놓친 경우입니다.
  • Point adjustment: contiguous anomaly segment 중 일부라도 맞추면 segment 전체를 검출로 보는 평가 보정 방식입니다.

실무에서는 Precision만 높다고 충분하지 않습니다. missed anomaly의 비용이 큰 domain에서는 Recall과 F1 해석이 더 중요해집니다.

Operations & deployment

논문 후반부 guideline를 이해하는 운영 용어

Real-time detection

이상이 생긴 즉시 잡아내는 전략입니다. latency와 false alarm cost가 중요합니다.

Early warning

실제 고장 전에 precursor를 찾아 경고하는 전략입니다. lead time과 missed anomaly trade-off가 핵심입니다.

Sliding window

고정 길이 segment로 시계열을 잘라 모델에 넣는 방식입니다. window size는 가장 중요한 hyper-parameter 중 하나입니다.

Incremental update

새 데이터가 들어올 때마다 상태를 조금씩 갱신하는 방식입니다. streaming 환경과 잘 맞습니다.

Batch learning

일정량의 데이터를 모아서 재학습하는 방식입니다. 재현성과 관리가 쉽지만 drift 대응은 느릴 수 있습니다.

Online update

운영 중 모델을 계속 적응시키는 전략입니다. drift에 강하지만 contamination과 forgetting 위험이 있습니다.

Denoising

노이즈를 줄여 false alarm을 낮추는 전처리입니다. 너무 강하면 약한 precursor까지 지울 수 있습니다.

Predictive maintenance (PdM)

설비가 실제로 고장 나기 전에 이상 신호를 이용해 maintenance timing을 앞당기는 전략입니다.

Root cause

이상 현상을 실제로 일으킨 원인 변수나 설비를 뜻합니다. graph-based relation modeling이 이 해석을 돕는 경우가 많습니다.
