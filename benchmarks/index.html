Benchmarks

SWaT · WADI · MSL을 어떻게 읽어야 하는가

이 논문은 public dataset 세 개를 통해 state-of-the-art model을 비교합니다. 중요한 건 “누가 이겼는가”보다 왜 특정 dataset에서 어떤 family가 강해졌는가를 이해하는 것입니다.

Paper takeaway: temporal dependency를 무시한 non-temporal baseline보다 sequence-aware model이 대체로 낫지만, dataset dimension과 relation strength에 따라 판도가 바뀝니다.

Result lens

논문은 F1-score를 baseline으로 정렬하되, Recall과 point adjustment가 실제 해석에 미치는 영향도 함께 설명합니다.

Dimension effect

WADI는 112 dimensions로 SWaT/MSL보다 훨씬 고차원이라 relation-aware modeling의 importance가 커집니다.

논문이 읽어낸 benchmark insight

  • Temporal modeling matters: DAGMM보다 RNN-based model이 SWaT/MSL에서 전반적으로 우세합니다.
  • Parallel processing for long sequence: Transformer/GTA는 long sequence를 병렬로 다루며 강점을 보입니다.
  • High dimension hurts naive reconstruction: WADI에서는 계산량과 feature explosion이 큰 병목입니다.
  • Graph relation can win: GDN/GTA는 attribute relation이 강한 setting에서 상대적으로 좋은 결과를 냅니다.

Precision / Recall / F1-score

실무에서는 false alarm도 비용이지만, missed anomaly는 더 큰 비용일 수 있습니다. 논문이 Recall과 F1을 더 중시한 이유입니다.

Point adjustment

contiguous anomaly segment 중 일부만 탐지해도 전체 segment를 detected로 간주해 recall/F1이 올라갈 수 있습니다.

Hyper-parameter effect

down-sampling, window size, learning rate, threshold 전략이 benchmark 해석 자체를 바꿀 수 있습니다.
